What it is

A personal, operator-run football-news studio

Football Flashing News is a server-side, headless content-automation tool built and run by a single operator. It generates original football graphics and publishes them to one Instagram account and one TikTok account that the operator owns. It is not a multi-user product: there are no sign-ups, no logins, no dashboards, and no ability for any third party to post through it.

🛠️ Single operator

One person builds the graphics and authorizes publishing. The tool only ever touches the single TikTok and single Instagram account the operator owns and explicitly connects.

⚽ Football-only content

Everything it publishes is original first-party football coverage — focused on the FIFA World Cup 2026 and the wider soccer calendar. No reposting of other people's media.

🔁 Mirrors Instagram to TikTok

The Instagram channel is already live. The TikTok integration mirrors that same media to TikTok so the two channels stay in sync, using TikTok's official API.

Original football graphics, produced in-house

Every post is a first-party graphic created by the service from cross-checked information. Four core formats make up the bulk of what goes out to Instagram and TikTok.

🏆 Match-result carousels

Multi-image carousels with final scores, scorers, key stats and penalty-shootout outcomes for World Cup 2026 and other fixtures.

Pre-match preview cards with line-up context, kickoff details and the storylines going into each game.

Clean transfer-update cards summarizing confirmed and reported moves across the football transfer market.

How it works

From graphic to published post

The pipeline is fully first-party: the service creates the media, verifies the facts, shows a full consent screen, and then publishes through TikTok's official API to the operator's own account.

1

Create the graphic

An original match-result carousel, reel, preview or transfer card is generated first-party — no scraped or third-party media.

2

Verify the facts

Scores, scorers and stats are cross-checked against two or more independent sources before anything is queued.

3

Review & consent

The operator reviews the post on a consent screen that shows the connected creator, an editable caption, a privacy selector and interaction controls.

4

Publish via official API

On confirmation, the post is published through TikTok's official Content Posting API (DIRECT_POST) to the operator's own authorized account.

TikTok integration

Built on TikTok's official Content Posting API

The TikTok connection uses the official Content Posting API over HTTPS, authorized through Login Kit OAuth. Video reels are published via DIRECT_POST using direct file upload (single or chunked). Photo carousels are published via DIRECT_POST with source=PULL_FROM_URL: TikTok server-side downloads each image from a public, ownership-verified HTTPS URL (photo posts also set auto_add_music as TikTok requires). Before any post, the service reads the account's creator_info and presents a complete content-sharing consent screen. The operator controls exactly what is published; publishing is disabled until a privacy level is chosen.

Pre-post consent screen — every element shown before publishing

  • ✓Creator identity — the connected account's nickname and avatar are shown at the top, read live from creator_info.
  • ✓Editable caption — a caption/title textbox the operator can edit before posting.
  • ✓Privacy selector with no default — the operator must actively pick a privacy level; nothing is pre-selected.
  • ✓Comment / Duet / Stitch controls — honoring creator_info; options the account has disabled are greyed out, none checked by default.
  • ✓Commercial-content disclosure — a toggle that, when on, reveals a Your Brand / Branded Content choice.
  • ✓Compliance declarations — dynamic Music-Usage confirmation, plus the Branded-Content policy statement when Branded is selected.
  • ✓Content preview — the reel plays or the carousel images are shown before publishing.
  • ✓Processing notice — a note that publishing may take a few minutes to process.
  • ✓Publish gating — the Post button stays disabled until a privacy level (and a commercial label, if commercial content is on) is chosen.
Pre-audit safety default: every post is forced to SELF_ONLY (private, visible only to the account owner). Public posting is only used after the Content Posting API audit passes.

Permissions & data

Exactly three scopes, and why each is needed

The integration requests only the scopes required to display the connected creator and publish the operator's own media. It accesses no other data — no followers, messages, comments, private content, or any other user's data.

ScopeWhy it is needed
user.info.basic Reads the authorized account's open ID, nickname/display name and avatar — used only to show the creator's identity on the consent screen and confirm the correct account is connected.
video.publish Publishes the operator's own media via DIRECT_POST — video reels by direct file upload, and photo carousels via source=PULL_FROM_URL.
video.upload Part of the same video.* product family that authorizes publishing the operator's first-party reels and photo carousels; it supports the inbox-upload path, where a video is sent to the account's TikTok inbox for the operator to finish and post from the TikTok app.

🔒 No third-party data

The service does not read or store any other TikTok user's data. It does not access followers, direct messages, comments or private content. OAuth tokens are stored locally on the operator's machine in a file excluded from version control, and are auto-refreshed. No end-user databases are maintained. Full detail is in the Privacy Policy.

↩️ Data deletion & revocation

The operator can revoke the service's access at any time from TikTok account settings (Security & permissions → Manage app permissions), which immediately invalidates the stored tokens. Deleting the local session file removes the stored access and refresh tokens. See the Privacy Policy for retention details.

FAQ

Frequently asked questions

Is this an official TikTok app or a TikTok partner?

No. Football Flashing News is an independent service that uses TikTok's official Content Posting API and Login Kit. It is not built, endorsed, or operated by TikTok, and it is not a TikTok partner.

Who can use it?

Only its single operator. There are no end-user accounts, logins, or dashboards. The service acts solely on behalf of its operator and only on the one TikTok account and one Instagram account the operator owns and authorizes.

What content does it post?

Original first-party football (soccer) graphics: match-result carousels, highlight and transfer reels, fixture previews, and transfer-news cards. It mirrors to TikTok the same content already posted to the brand's Instagram account.

How is account data used?

The connected account's basic info (open ID, nickname, avatar) is used only to display the creator on the consent screen and confirm the right account is connected. Access tokens are used solely to publish the operator's own content. The service does not use any data for advertising, profiling, resale, or any unrelated purpose, and does not read followers, messages, comments, or private content.

How do I revoke access?

Revoke the service's access at any time from your TikTok account settings under Security & permissions → Manage app permissions. That immediately invalidates the stored tokens. Removing the local session file also deletes the stored access and refresh tokens.

Is the content original?

Yes. The graphics are produced first-party by the service from cross-checked information. Results and stats are verified against at least two independent sources before publishing. Content is provided on a reasonable-efforts basis and is not guaranteed.
